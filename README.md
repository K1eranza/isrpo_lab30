## Таблица атрибутов JSON

| Атрибут / настройка | Что делает | Пример |
|---------------------|------------|--------|
| `[JsonPropertyName("name")]` | Задаёт имя поля в JSON | `"name"` вместо `"Name"` |
| `[JsonIgnore]` | Поле не попадает в JSON | Пароли, внутренние заметки |
| `PropertyNamingPolicy.CamelCase` | Все поля в camelCase | `"realName"` вместо `"RealName"` |
| `JsonStringEnumConverter` | Enum как строка | `"Marvel"` вместо `0` |
| `WriteIndented = true` | JSON с отступами | Для удобства при разработке |

## Главные выводы

1. **Сериализация** — это процесс преобразования C#-объекта в JSON-строку для передачи по сети. **Десериализация** — обратное преобразование JSON-строки в C#-объект.

2. **`[JsonIgnore]`** нужен для исключения полей из JSON-ответа. Например, поле `InternalNotes` (внутренние заметки разработчика) или пароли не должны попадать клиенту — это защищает данные от утечки.

3. **Enum лучше сериализовать строкой**, а не числом, потому что клиенту сразу понятно значение: `"universe": "Marvel"` читается легче, чем `"universe": 0`. Это делает API более понятным и самодокументируемым.

4. **После настройки CamelCase** в ответе API все поля стали с маленькой буквы: `realName` вместо `RealName`, `powerLevel` вместо `PowerLevel`. Это соответствует общепринятому стилю именования в JSON и JavaScript, что упрощает работу фронтенда.
